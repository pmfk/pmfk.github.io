# vuepress 博客搭建并部署到 github pages

# 前言

作为一个产品经理，怎能没有自己的博客？

之前一直用 WordPress 搭建博客，但是自从网站被黑过一次后，就转向使用无后端的纯静态网站，例如非常受欢迎的 hexo 。

想比于其它老牌的静态网站生成器，vuepress 这个项目非常年轻，是2018年才由 Vue 的作者启动的。

vuepress 简洁优雅，配置简单，开箱即用，非常适合写技术和产品文档。

作为一个产品经理，怎能没有探究新事物的好奇心？

本文记录了搭建 vuepress 博客以及相关配置的流程，希望能够有帮助。

# github 准备

  1. 准备一个邮箱，例如 test@qq.com 。
  2. 用邮箱注册 github，用户名随便取，例如 test 。
  3. 创建一个github仓库，点击 new repository ，init README 那里勾上，仓库命名为 test.github.io 。注意仓库名一定要是用户名后边加上 .github.io 才行。

仓库创建完成后，可以看到有一个README.md文件。

# GitHub pages 设置

  1. 点击仓库 test.github.io 的 setting，找到GitHub Pages 这部分，点击 none ，下拉选中master branch，保存。

完成后，你将会看到绿色的状态栏提示，点击链接，就可以看到自己的网站了。

现在你可以用 test.github.io 这个网址访问你的网站了。

# vps 准备 (可选)

vps准备

相对于本地搭建，我更偏向于在 vultr 的 vps 上，这样可以避免软件被墙导致的的各种问题。

我使用的系统是 linux centos 7。

vuepress 依赖git，没安装过需要安装下。

  1. yum install git -y

配置Git

  1. ssh-keygen -t rsa -C test@qq.com，邮箱换成你注册GitHub的邮箱。一路敲回车就ok
  2. cat .ssh/id_rsa.pub
  3. 复制显示的那一大串内容，注意不要带多余的字符。
  4. 来到GitHub的 test.github.io 仓库，点击 setting，点击左侧的 deploy keys，点击 add deploy key，title 名字随便取，例如 vuesasaki，将上一步中复制的内容粘贴到 key 框框里。allow access 勾选。点击 add key 。
  5. 回到终端，git config --global user.name "test"，注意将 test 换成你的 GitHub 的用户名
  6. git config --global user.email "test@qq.com"，注意将 test@qq.com 换成你的GitHub注册邮箱。

Git远程下载测试

  1. 来到你的test.github.io仓库，点击绿色的clone or download，复制URL地址
  2. 回到终端，git clone 地址，将地址换成你复制的仓库地址。
  3. ls

若发现了test.github.io文件夹，说明远程下载成功。

设置Git推送免输入密码

  1. cd test.github.io，将test.github.io换成你的文件名。
  2. git remote -v
  3. git remote set-url origin git@github.com:test/test.github.io
  4. git push -u origin master

注意将第3步中，/ 前边的 test 换成你的GitHub用户名，后边的 test.github.io 换成你的GitHub仓库。

Git远程推送测试

  1. git rm README.md
  2. git add .
  3. git commit -m 'delete README.md'
  4. git push

回到GitHub仓库页面，刷新，若README.md文件消失，说明Git远程push成功。

# 安装 node.js

使用 nvm 来安装 node.js ，它可以安装和管理多个版本的 node，就像在同一台机器上安装 Python2 和 Python3 一样。

  1. curl https://raw.github.com/creationix/nvm/master/install.sh | sh
  2. wget -qO- https://raw.github.com/creationix/nvm/master/install.sh | sh

重启终端

退出终端，重新登录进来。

  1. exit
  2. ssh root@111.111.111.111

把111.111.111.111换成你自己 vps 的 IP 地址，输入密码，登录

安装

  1. nvm install v9.0.0

注意这里不能安装稳定的新版本 nvm install stable，否则安装 vuepress 时会报错，而 vuepress 又需要 node 8 以上的版本，所以我们安装 node 9 就可以了。

# 安装 vuepress 搭博客

安装vuepress

vuepress 1.0 版本正处于 alpha 阶段，可能会有很多 bug，所以我们安装稳定的旧版本，等 vuepress 正式发布后再进行升级。

  1. npm install -g vuepress

创建博客

  1. mkdir vue
  2. cd vue
  3. echo '# i love vue' > README.md

将vue换成你自己想取的目录名，随便取。

查看效果

  1. vuepress dev

打开浏览器，输入http://111.111.111.111:8080/。把111.111.111.111换成你的vps的IP地址。

看到 i love vue 网页，成功！

生成静态文件

  1. 键盘按下Ctrl + c，回到终端。
  2. vuepress build

部署到GitHub pages

  1. cp -r .vuepress/dist/* ~/test.github.io，注意将test.github.io换成你下载下来的文件名，也就是你的GitHub仓库名。
  2. cd ~/test.github.io
  3. git add .
  4. git commit -m 'blog'
  5. git push

打开浏览器输入test.github.io，发现你的vuepress博客已经推送到 GitHub pages上了。

# vuepress 页面

简述vuepress网页生成的原理

一个以 .md 为后缀的文件可以生成一个 .html 网页。这个转换过程，由vuepress自动帮我们完成，不需要我们动手。

vuepress总是将一个 README.md 文件转换成名为 index.html 的网页，作为整个网站或者单个栏目的首页。

我们建立的那个写有 i love vue 的README.md 文件，就被当做整个网站的首页。

而单个栏目的首页又是什么？别急，接下来你就清楚了。

通常情况下，一个完整的网站，不止有一个网页。所以，我们要创建更多的.md文件，让vuepress生成更多的.html网页。

网页之间，就是用我们熟悉的链接来把它们关联在一起。

# 创建新的页面

这里用到编辑器 vim ，对新手来说它的学习曲线有些陡峭。

  1. vim about.md。用vim编辑器创建并编辑一个新的.md文件
  2. 在打开的文件中，按下键盘的 i，注意一定要是英文小写字母 i
  3. 输入 # 我的个人简介
  4. 按下键盘左上角的 esc 键。
  5. 确保输入法为英文状态，输入一个 英文冒号 ，也就是按键盘上的 shift + : 。这时你会发现终端上的白色光标，已经来到了左下角。
  6. 输入英文小写字母 x，然后敲回车。
  7. ls

这时你会发现，你退出了编辑器，并且看到了新建的 about.md 文件。

# 链接两个Markdown文件

  1. vim README.md。用 vim 编辑器打开 README.md 文件。
  2. 按下键盘上的 A 。注意是英文状态下的大写字母 A 。你会发现，光标来到了本行的末尾。
  3. 敲几下回车。目的是增加几个空行。
  4. 输入 [点我](about.md)
  5. 按下键盘左上角的 esc 键。
  6. 确保输入法为英文状态下的输入。输入一个 英文冒号 ，也就是按键盘上的 shift + : 。这时你会发现终端上的白色光标，已经来到了左下角。
  7. 输入英文小写字母 x，然后敲回车。

通过以上步骤，你新建了一个 about.md 文件，并且在里边输入了一行内容。随后在 README.md 文件中，你加入了对新文件的 链接 。

# 查看链接效果

  1. vuepress dev
  2. 打开浏览器输入 http://111.111.111.111:8080/

注意将111.111.111.111换成你自己vps的IP地址。

端口号有时会变化，并不总是 8080 。例如，本次我再输入 vuepress dev 命令后，vuepress给我指定了 8082 端口号。

 DONE  [03:44:16] Build 38a381 finished in 6178 ms! 

 VuePress dev server listening at http://localhost:8082/

总之，当你输入 vuepress dev 命令后，按照输出的信息，将 localhost 换成你的vps的IP地址，将冒号后边的端口号换成vuepress指定的端口。

浏览器打开网站后，你会发现首页中多了一个绿色的链接。

点击链接，它会把你带到另外一个页面。

这个新网页就是新建立的 about.md 文件转换生成的，你从浏览器的地址栏可以看到 about.html 。

# 建立一个栏目

栏目的用途

在上一步中，你学会了创建一个.md文件，以此生成对应的网页。

如同之前所说，网站中可能有很多网页。

所以，你可能会创建很多.md文件，用来生成很多的网页。

并且可以按照上一步中学到的，依靠在 README.md 文件中加入链接的方式，在网站首页加入一个个新网页的链接。

这种方式有个弊端。当你网站的页面足够多以后，你如果要寻找某篇文章，就只能在首页寻找指向那篇文章的链接。寻找信息的效率会很低。

如果你的网站中许多网页内容相关，而不是一个类似于 about.html 这个孤立的自我介绍的页面，可以想象，将那些内容相关的网页归类放在同一个地方，查找信息的效率将大大增高。

例如，你有很多日语学习笔记的网页，和学习编程网页。

这种情景下，将日语学习笔记相关网页放到一起，将编程的网页放到一起，将会大大方便你查找日语或编程内容的效率。

网站栏目就是这样用的。

# 建立栏目目录

  1. Ctrl + c
  2. mkdir japanese

我们以后就把所有日语相关的.md文件 ，都放到 japanese 这个文件夹中。

  1. mkdir code

类似的，可以把所有编程相关的.md文件 ，都放到 code 这个文件夹中。

# 建立栏目的首页

  1. echo '# 这是日语栏目的首页' > ./japanese/README.md
  2. echo '# 这是编程栏目的首页' > ./code/README.md

栏目的首页文件和网站的首页文件一样，都必须命名为 README.md 。

# 查看网站目录结构

  1. yum install tree -y
  2. tree

你会看到如下的树形目录结构。

.
├── about.md
├── code
│   └── README.md
├── japanese
│   └── README.md
└── README.md

正如你看到的，整个网站的首页文件，和两个栏目的首页文件，名字都是 README.md 。

在栏目中多建一个文件

  1. echo '# 日语语法页面' > ./japanese/grammar.md
  2. echo '# python 编程学习页面' > ./code/python.md

查看目录

  1. tree

结构如下：

.
├── about.md
├── code
│   ├── python.md
│   └── README.md
├── japanese
│   ├── grammar.md
│   └── README.md
└── README.md

# 查看栏目页效果

  1. vuepress dev
  2. 打开浏览器输入 http://111.111.111.111:8080/code/。IP和端口号自己替换。

这样你就能访问code栏目的首页了。

输入网站 http://111.111.111.111:8080/code/python.html 并回车，这样你就能访问code栏目中的python页面了。

Japanese这个栏目的首页，和栏目下边的子页面，跟code一样，通过更改相应的地址，就可以访问了。

但是，你可能会问，这个访问方式还是很麻烦。

哈哈，接下来，等我们配置好网站的导航栏和侧边栏后，问题就迎刃而解了！

# vuepress配置文件

  1. Ctrl + c
  2. vim .vuepress/config.js
  3. 在打开的文件中，按下键盘的 i，注意一定要是英文小写字母 i
  4. 输入以下内容：
module.exports = {
  title: 'hello vue',
  description: 'vue is kawaii'
}

保存内容退出vim

步骤跟上文中的退出vim都是一样的。

  1. 按下键盘左上角的 esc 键。
  2. 确保输入法为英文状态，输入一个 英文冒号 ，也就是按键盘上的 shift + : 。这时你会发现终端上的白色光标，已经来到了左下角。
  3. 输入英文小写字母 x，然后敲回车。

# 查看网站title配置效果

  1. vuepress dev
  2. 打开浏览器，输入相应的网址，查看网页

可以看到，网页导航栏最左边出现了 hello vue，这是我们上一步在config.js里边设置的网站标题。它始终会出现在导航栏的最左边。

同时，你也可以看到浏览器的tab上，出现了i love vue | hello vue。

# vuepress配置导航栏

  1. 按下Ctrl + c，回到终端。
  2. vim .vuepress/config.js
  3. 在打开的文件中，按下键盘的 i，注意一定要是英文小写字母 i
  4. 在大括号中，插入以下配置nav（导航栏）的内容：
themeConfig: {
    nav: [
      { text: '关于我', link: '/about' },
      { text: '日语', link: '/japanese/' },
      { text: '编程', link: '/code/' },    
    
      
    ]
  }

保存内容退出vim

步骤跟上文中的退出vim都是一样的。

  1. 按下键盘左上角的 esc 键。
  2. 确保输入法为英文状态，输入一个 英文冒号 ，也就是按键盘上的 shift + : 。这时你会发现终端上的白色光标，已经来到了左下角。
  3. 输入英文小写字母 x，然后敲回车。

确保你的config.js文件里，最终的内容是像这样的：

module.exports = {
  title: 'hello vue',
  description: 'vue is kawaii',

themeConfig: {
    nav: [
      { text: '关于我', link: '/about' },
      { text: '日语', link: '/japanese/' },
      { text: '编程', link: '/code/' },    
      
      ]
  }


}

# 配置侧边栏

在完成上一步后，你如果点击导航栏中的日语和编程链接，会来到两个栏目的首页。

而两个栏目中，除了README.md生成的栏目首页，还有python.md和grammar.md这两个文件生成的网页，怎么找不到它们呢，它们在哪里？

别急，把侧边栏配置一下，就能看见它们了！

  1. 按下Ctrl + c，回到终端。
  2. vim .vuepress/config.js
  3. 在打开的文件中，按下键盘的 i，注意一定要是英文小写字母 i
  4. 在大括号中，插入以下配置sidebar（侧边栏）内容：
sidebar: {
      '/japanese/': [
        '',
        'grammar',
      ],

      '/code/': [
        '',
        'python',
      ],

         '/': [
         '',
        'about',
       ]
}

保存内容退出vim

步骤跟上文中的退出vim都是一样的。

  1. 按下键盘左上角的 esc 键。
  2. 确保输入法为英文状态，输入一个 英文冒号 ，也就是按键盘上的 shift + : 。这时你会发现终端上的白色光标，已经来到了左下角。
  3. 输入英文小写字母 x，然后敲回车。

确保你的config.js文件里，最终的内容是像这样的：

module.exports = {
  title: 'hello vue',
  description: 'vue is kawaii',

themeConfig: {
    nav: [
      { text: '关于我', link: '/about' },
      { text: '日语', link: '/japanese/' },
      { text: '编程', link: '/code/' },    
    ],

sidebar: {
      '/japanese/': [
        '',
        'grammar',
      ],

      '/code/': [
        '',
        'python',
      ],

         '/': [
         '',
        'about',
       ]
}
  }
}

不要少了或者错了各种符号，例如逗号，大括号等。建议删除原本配置的全部内容，直接复制粘贴这段代码。

# 查看侧边栏配置效果

  1. vuepress dev
  2. 打开浏览器，输入相应的网址，查看网页

点击日语导航链接后，可以在日语栏目首页看到左侧的侧边栏，而grammar.md生成的页面，也列在了侧边栏。

同时，vuepress自动帮我们生成了类似上一页和下一页的链接。点击它们就可以翻页。

你可以去编程栏目看看，也是一样的效果。

这样，我们把内容相关的网页归档在一起的目的就实现了！
